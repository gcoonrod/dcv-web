# Data Model: Lemon Squeezy Payment Integration

**Feature**: 005-lemonsqueezy-payments
**Date**: 2026-03-11

---

## Overview

This integration is entirely client-side and static. There is no persistent data store on dcv-web. All order, license, and customer data is owned and stored by Lemon Squeezy. The entities below describe the logical data concepts that the integration depends on, not server-side models.

---

## Entities

### 1. Product Variant

The specific purchasable SKU configured in the Lemon Squeezy dashboard representing the DCV Pro perpetual license.

| Attribute | Type | Description |
|-----------|------|-------------|
| `variant_id` | string | Unique LS variant identifier; used to construct the checkout URL |
| `checkout_url` | string | Full Lemon Squeezy hosted checkout URL for this variant (e.g., `https://gcoonrod.lemonsqueezy.com/buy/{variant_id}`) |
| `price_display` | string | Human-readable price string hardcoded in the Astro component (e.g., `"$49"`) |
| `product_name` | string | `"DCV Pro"` — used for display and receipt labeling |
| `license_type` | string | `"perpetual"` — covers all v1.x.x minor updates and patches |

**Source of truth**: Lemon Squeezy dashboard. The `checkout_url` and `price_display` are the only values referenced by the static site.

**Validation rules**:
- `checkout_url` MUST be a valid HTTPS URL pointing to `*.lemonsqueezy.com`
- `price_display` MUST NOT be the placeholder string `"[PRICE]"` at deploy time

---

### 2. Checkout Session (external — LS-managed)

Created implicitly by Lemon Squeezy when the customer opens the checkout overlay. Not controlled or stored by dcv-web.

| Attribute | Type | Description |
|-----------|------|-------------|
| `customer_email` | string | Entered by customer during checkout; used for license delivery |
| `payment_status` | enum | `pending` → `paid` or `failed` (managed by LS) |
| `discount_code` | string? | Optional promo code applied by customer |
| `tax_amount` | number | Calculated by LS based on customer location |

**Constraints**:
- dcv-web has no read or write access to checkout session data
- Discount codes are configured and validated entirely in the LS dashboard

---

### 3. Order (external — LS-managed)

Created by Lemon Squeezy upon successful payment. The `order_id` is available as a URL parameter on the success page if LS link variables are configured.

| Attribute | Type | Description |
|-----------|------|-------------|
| `order_id` | string | LS-assigned unique order identifier |
| `customer_email` | string | Email address to which receipt and license key are sent |
| `total_formatted` | string | Formatted total paid (e.g., `"$49.00"`) |
| `created_at` | datetime | UTC timestamp of order creation |

---

### 4. License Key (external — LS-managed)

Generated by Lemon Squeezy per order, delivered via email. Referenced on the `/dcv/success` page as a static command example.

| Attribute | Type | Description |
|-----------|------|-------------|
| `key` | string | Unique alphanumeric key (e.g., `XXXX-XXXX-XXXX-XXXX`) |
| `order_id` | string | Foreign key to the Order that generated this key |
| `activation_limit` | number | Maximum number of machines this key can activate (configured in LS dashboard) |
| `status` | enum | `inactive` → `active` (managed by LS / DCV CLI) |

**Key delivery**: Via Lemon Squeezy automated email only. The dcv-web success page does NOT read or display the customer's actual license key — it shows only a static example command (`dcv activate <your-key>`).

---

### 5. Success Page State (client-side only)

The `/dcv/success` Astro page is fully static. It has no runtime state — all content is hardcoded at build time.

| Content Element | Value |
|-----------------|-------|
| Confirmation message | `"Your purchase was successful! Check your email for your license key."` |
| Activation command | `dcv activate <your-license-key>` (static code block) |
| Docs link | `${BASE_URL}/docs/installation` |

---

### 6. Checkout Button State (client-side runtime)

The `CheckoutButton` component manages two runtime states in the browser:

| State | Trigger | Button appearance |
|-------|---------|-------------------|
| `ready` | LS script loaded successfully | Enabled CTA — `"Buy Pro"` |
| `unavailable` | LS script `onerror` fired | Disabled — `"Checkout temporarily unavailable"` |

**Default (no JS)**: Plain `<a href="checkout_url">` link — routes to LS hosted checkout page.

---

## State Transitions

```
Page load (no JS)
  └─► Button = plain link to LS checkout URL [Constitution I satisfied]

Page load (JS enabled)
  └─► Load lemon.js script
        ├─► script.onload ─► LS auto-initializes overlay on .lemonsqueezy-button
        │                      └─► Button = overlay trigger (FR-007)
        └─► script.onerror ─► Button = disabled + "unavailable" message (FR-009)
```

## Configuration Constants

These values are set at build time in the Astro source and are not computed at runtime:

| Constant | Location | Example value |
|----------|----------|---------------|
| `CHECKOUT_URL` | `src/pages/index.astro` (passed as `checkoutUrl` prop → `Pricing.astro` → `CheckoutButton.astro`) | `https://gcoonrod.lemonsqueezy.com/buy/{variant}` |
| `PRO_PRICE` | `src/pages/index.astro` (prop) | `"$49"` |
| `SUCCESS_URL` | Lemon Squeezy dashboard | `https://apps.microcode.io/dcv/success` |
