# Implementation Plan: Docs, Installation, & Quickstart (Phase 3)

**Branch**: `003-docs-and-quickstart` | **Date**: 2026-03-06 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/003-docs-and-quickstart/spec.md`

## Summary

Implement a static documentation portal using Astro 5's Content Layer API and `@tailwindcss/typography`. Four Markdown files in `src/content/docs/` are automatically routed via `[...slug].astro` with a responsive two-column `DocsLayout` (sidebar + content). Sidebar collapses above content on mobile via CSS-only `flex-col`. No client-side JavaScript is introduced. One new dependency (`@tailwindcss/typography`) is added.

## Technical Context

**Language/Version**: TypeScript (strict, Astro default) — Node.js 20 LTS
**Primary Dependencies**: Astro 5.18.x, `@tailwindcss/vite` (Tailwind CSS v4), `@tailwindcss/typography` (new)
**Storage**: File-based — Markdown in `src/content/docs/`, validated at build time via Zod
**Testing**: Manual browser testing per quickstart.md — no automated test suite (not requested)
**Target Platform**: Static HTML/CSS — AWS S3 + CloudFront; `output: 'static'`
**Performance Goals**: Lighthouse 90+ on docs pages; zero new JS bundles
**Constraints**: Zero client-side JS on docs pages; `output: 'static'` unchanged; Tailwind v4 `@plugin` syntax (no JS config)
**Scale/Scope**: 4 doc pages for Phase 3; designed to scale to N pages via Content Layer glob loader with zero routing changes

## Implementation Steps

### Step 1: Install Dependency

```bash
npm install @tailwindcss/typography
```

### Step 2: Register Typography Plugin (global.css)

Add `@plugin "@tailwindcss/typography"` after `@import "tailwindcss"` in `src/styles/global.css`. This is the Tailwind v4 plugin registration syntax — no `tailwind.config.js` required.

### Step 3: Configure Shiki Theme (astro.config.mjs)

Add `markdown: { shikiConfig: { theme: 'github-dark' } }` to `defineConfig`. Shiki is bundled in Astro 5 — no package install needed.

### Step 4: Define Content Collection (src/content/config.ts)

Create `src/content/config.ts` using the Astro 5 Content Layer API (`glob` loader from `astro/loaders`). Zod schema requires `title: z.string()`, `description: z.string()`, `order: z.number()`.

### Step 5: Create Baseline Markdown Files (src/content/docs/)

Create four files: `installation.md`, `quickstart.md`, `configuration.md`, `changelog.md`. Each file must include all three required frontmatter fields. See `data-model.md §Content Outline` for exact content.

### Step 6: Create DocsSidebar.astro (src/components/)

Accepts `entries` and `currentSlug` props. Renders `<nav aria-label="Documentation navigation">` with `<ul>/<li>/<a>` structure. Applies `text-dcv-accent font-bold` to the active link. Sorts by `entry.data.order`.

### Step 7: Create DocsLayout.astro (src/layouts/)

Wraps the global `Layout`. DocsLayout MUST fetch and sort all doc entries in its own frontmatter block (`getCollection('docs')`, sorted by `order`) and pass them to `DocsSidebar`. Composes `<aside>` (DocsSidebar) + `<div>` (content column — NOT `<main>`, since Layout.astro already provides `<main>`) in a `flex flex-col md:flex-row` container. Sidebar is `md:sticky md:top-0 md:h-screen md:overflow-y-auto`. Content area applies `prose prose-invert max-w-none prose-pre:overflow-x-auto`. Pass both `title` and `description` through to `<Layout>`. See `contracts/component-api.md §Contract 3` for the complete required frontmatter block and output structure.

### Step 8: Create [...slug].astro (src/pages/docs/)

Uses `getStaticPaths` with `getCollection('docs')`. Maps `entry.id` to `params.slug`. Renders `<DocsLayout>` with `<Content />` slot.

### Step 9: Update Layout.astro (Add Global Header + Docs Nav Link)

**Note**: The current `Layout.astro` has NO `<header>` element — it is a bare `<body><main><slot /></main></body>` shell. This step creates a `<header>` from scratch immediately before `<main>`. Add a `<header class="border-b border-slate-800 px-6 py-4">` containing a `<nav>` with a site logo link (`href="/"`) and a "Docs" link (`href="/docs/installation"`). See `contracts/component-api.md §Contract 5` for the exact required markup including all Tailwind classes.

### Step 10: Build Verification

Run `npm run build`. Verify:
- `dist/docs/installation/index.html` exists with curl command
- `dist/docs/quickstart/index.html` exists
- `dist/docs/configuration/index.html` exists
- `dist/docs/changelog/index.html` exists
- No new `.js` files in `dist/_astro/` (only CSS hash change expected)

## Design Decisions Reference

All key decisions are documented in [`research.md`](./research.md):

- **D1**: `@tailwindcss/typography` via `@plugin` directive (Tailwind v4 compatible)
- **D2**: Content Layer API (`loader: glob`) — not legacy `type: 'content'`
- **D3**: Active state via `currentSlug` prop (build-time, zero JS)
- **D4**: Mobile layout via `flex-col md:flex-row` (CSS-only, confirmed by clarification)
- **D5**: `github-dark` Shiki theme
- **D6**: `md:sticky md:top-0 md:h-screen md:overflow-y-auto` for sidebar scroll independence
- **D7**: `/docs` bare route → 404 acceptable (confirmed by clarification)
