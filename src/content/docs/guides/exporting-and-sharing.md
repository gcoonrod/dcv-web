# Exporting & Sharing

This guide covers converting your dcv reports into formats suitable for sharing with managers, uploading to HR systems, or printing.

## Export Formats

Reports generated by `dcv analyze` are written as Markdown by default. The `dcv export` command converts them into other formats:

| Format   | Flag             | Use Case                                      |
|----------|------------------|-----------------------------------------------|
| Markdown | *(default)*      | Generated by `analyze`, editable in any editor |
| HTML     | `--format html`  | Self-contained, styled page for sharing        |
| LaTeX    | `--format tex`   | Compile to PDF with `pdflatex` or Overleaf    |

## Exporting to HTML

Generate a self-contained HTML file from your Markdown report. You can reference a report by full path or by its shorthand ID (the filename without the `.md` extension, resolved from `~/.dcv/reports/`):

```bash
# Using a full path
dcv export ~/.dcv/reports/dcv_report_20260301.md --format html

# Using shorthand ID (resolved from ~/.dcv/reports/)
dcv export dcv_report_20260301 --format html
```

Or specify a custom output path with `-o`:

```bash
dcv export dcv_report_20260301 --format html -o ~/Desktop/review.html
```

The HTML export includes inline styles, so the file renders correctly when opened in any browser or shared as an email attachment. No external stylesheets or assets are required.

This is the fastest way to produce a polished, shareable version of your report.

## Exporting to LaTeX

Generate a LaTeX file for high-quality PDF output:

```bash
dcv export ~/.dcv/reports/dcv_report_20260301.md --format tex -o ~/review.tex
```

Once exported, compile to PDF locally:

```bash
pdflatex ~/review.tex
```

Alternatively, upload the `.tex` file to [Overleaf](https://www.overleaf.com) for online editing and PDF generation without a local LaTeX installation.

LaTeX export is ideal when you need precise typographic control or when your organization requires PDF submissions.

## Locating Reports

Reports generated by `dcv analyze` are saved to `~/.dcv/reports/` by default. You can also specify a custom output path:

```bash
dcv analyze --since 2026-01-01 --until 2026-03-31 -o ~/Desktop/q1_review.md
```

When exporting, you can reference reports by:

- **Full path**: `dcv export ~/Desktop/q1_review.md --format html`
- **Shorthand ID**: `dcv export dcv_report_20260301 --format html` (looks for `~/.dcv/reports/dcv_report_20260301.md`)

To export with a custom output path, use the `-o` flag:

```bash
dcv export dcv_report_20260301 --format html -o ~/Desktop/polished_review.html
```

If you don't specify `-o`, the export file is written to the current directory with an appropriate extension.
